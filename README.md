# DormFix

## Project Overview

Dormitory residents often experience maintenance issues such as plumbing problems, electrical faults, or damaged facilities. Traditionally, these issues are reported manually, which can cause delays, miscommunication, and lack of tracking.

The DormFix System aims to digitize and streamline the maintenance reporting process through a centralized mobile and web application. The system allows residents to submit maintenance requests, track their status, and receive updates. Maintenance staff can review, update, and resolve requests efficiently within the platform.


---

## Key Features

### 🔐 Authentication System
- Secure user registration and login
- Role-based access control (Resident, Maintenance Staff)
- JWT-based authentication

### 🔄 Maintenance Workflow Features
- Residents can submit maintenance requests
- Residents can view and track request status (Pending, In Progress, Completed)
- Maintenance staff can update request status
- Staff can add comments or updates to requests
- Residents can view staff remarks

### 🔔 Notifications
- In-app notifications for status updates
- Alerts when a request is approved, updated, or completed

### 🔧 RESTful APIs
- Endpoints for user authentication
- Endpoints for maintenance request submission and management
- Endpoints for status updates and comments
- Consistent error handling responses
- API testing using Postman

### 🖥 Frontend UI
- Dashboard for residents showing submitted requests
- Staff dashboard for managing and filtering requests
- Status indicators with visual tags
- Request detail page with comments section
- Profile management page

### 🗃 Database Design
- Schema for users, maintenance requests, comments, and notifications
- Foreign key relationships between users and requests
- Secure password storage (hashed)
- Proper indexing for performance optimization
